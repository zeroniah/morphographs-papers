## Citation Strategy: The "Progressive Enhancement" Model

### Phase 1: Initial Upload (v1 - Week 1)

**Placeholder Citation Format:**
```
For related analysis, see Hesse (2026), "[Paper Title]," available on SocArXiv 
(DOI: pending, https://socarxiv.org/preprints/XXXXX).
```

**What gets uploaded:**
- P005 OR P001 (whichever is more polished)
- All other papers reference this as "forthcoming"

### Phase 2: Cascade Update (v2 - Week 2)

**Once P005/P001 has DOI:**
```
For medical term stability analysis, see Hesse (2026), "When Blood Spoke the 
Same Language," SocArXiv, https://doi.org/10.31235/osf.io/XXXXX.
```

**What gets updated:**
- All 7 remaining papers cite P005/P001 with real DOI
- Upload v1 of remaining papers

### Phase 3: Closed Loop (v3 - Week 3-4)

**All papers cite each other:**
- Update all papers with final DOIs
- Upload v2/v3 (SocArXiv preserves DOI across versions)
- WordPress posts link to final DOI versions

## Implementation Script Ideas

### Script 1: `19_build_metadata_registry.py`
**Purpose:** Auto-generate citation strings from this registry

**Input:** This markdown file  
**Output:** JSON with ready-to-paste citation strings

**Example output:**
```json
{
  "P005": {
    "doi": "10.31235/osf.io/abc123",
    "short_cite": "Hesse (2026)",
    "full_cite": "Hesse, N. (2026). When blood spoke...",
    "status": "published"
  }
}
```

### Script 2: `20_update_cross_references.py`
**Purpose:** Batch-update all blog posts with new DOIs

**Logic:**
1. Read registry JSON
2. Find placeholder text: `(DOI: pending)`
3. Replace with real DOI from registry
4. Generate v02 files with updated citations

## WordPress Integration Notes

**Custom Field:** `paper_doi`  
**Why:** Allows you to update DOI without re-editing entire post

**Template logic:**
```php
<?php if (get_field('paper_doi')): ?>
  <p class="citation">
    Cite this post: <?php echo get_field('full_citation'); ?> 
    <a href="https://doi.org/<?php echo get_field('paper_doi'); ?>">
      DOI: <?php echo get_field('paper_doi'); ?>
    </a>
  </p>
<?php endif; ?>
```
